Package: com.disney.studio.qa.stbx.pageobject

[Groovy] Class Page

    • com.disney.studio.qa.stbx.pageobject.Page

    • The Page class encapsulates a set of behaviors that aid in using Selenium by formalizing methods around accessing UI elements. In addition to this, methods exist that support more common tasks such as retrieving the page title, refreshing the page or waiting for an element to be visible.

      Traditional Selenium

      Typically, when one follows the Page Object Patten they do so by creating a series of page object classes that implement one or more services that allow Selenium to interact with elements that reside on that page. For instance, using the example presented by the Selenium group, if one was on the login page and needed to enter a username and password, the implementation looks like 

       // The login page contains several HTML elements that will be represented as WebElements.
 // The locators for these elements should only be defined once.
 By usernameLocator = By.id("username");
 By passwordLocator = By.id("passwd");
 By loginButtonLocator = By.id("login");

 // The login page allows the user to type their username into the username field
 public LoginPage typeUsername(String username) {
   // This is the only place that "knows" how to enter a username
   driver.findElement(usernameLocator).sendKeys(username);

   // Return the current page object as this action doesn't navigate to a page represented by another Page
   return this;
 }

 // The login page allows the user to type their password into the password field
 public LoginPage typePassword(String password) {
   // This is the only place that "knows" how to enter a password
   driver.findElement(passwordLocator).sendKeys(password);

   // Return the current page object as this action doesn't navigate to a page represented by another Page
   return this;
 }

      Using the Selenium Toolbox API

      Now, instead of this ... when using the Page class one could do something like 

       class LoginPage extends Page {
   private By passwordLocator = By.id('passwd');

   public LoginPage(WebDriver driver) {
      super(driver);
   }
   LoginPage typePassword(String password) {
      sendText(passwordLocator, password);
      return this;
   }
 }

      The benefit of using sendText(), instead of a more traditional approach, resides in the fact that before any text is entered into the text field, the framework waits until the web element (in this case, the text field), is present and visible on the page. How is this possible? Well, the sendText() method has logic to wait until the specified locator is present and visible on the page. The same can be said for any of the other Html5Accessor methods. This alleviates the need to populate your code with sleep conditions and/or other types of wait conditions.

      Choosing the appropriate Accessor

      Selenium Toolbox supports the use of Accessors to interact with some of the more common web elements found on a page. As of this writing, there are 3 types of accessors supported by the framework: Html5Accessor and ExtJSAccessor. Choosing which accessor to use depends upon the kind of web page you might be dealing with. For instance, let's assume that the login page used in the above example contains web elements based on HTML5 standards. To interact with the page using Html5Accessors, simply create a LoginPage by supplying it with an instance of Selenium's WebDriver. If an accessor is not explicitly defined, then the default behavior is to use the Html5Accessor. 

       LoginPage loginPage = new LoginPage(driver);

      The use of accessors like the Html5Accessor is based upon Groovy's @Delegate annotation. The Page class delegates responsibility of the web controls (e.g., buttons, links, text fields, etc.) over to the appropriate Accessor class. Essentially, this integrates Accessor behavior into the Page class

      See Practically Groovy: The Delegate annotation for a nice write-up on the @Delegate annotation

        • long explicitGlobalTimeoutInSeconds

          The timeout in seconds to wait. This is used in methods like isElementPresentAndVisible(By), isElementPresent(By), waitUntilElementIsPresent(By), waitUntilEachElementIsPresent(By), etc. Defaults to 10 seconds. This setting can be overridden. However, keep in mind, if the setting is overridden, because it is a GLOBAL variable, it will affect all timeout values associated with the waitUntil*, isElement* and isEachElement* methods
